Промпт для разработки клиентского фронтенда CRM, учитывающий все новые вводные и референс с GitHub:

Задача: Разработать клиентский фронтенд для CRM-системы — публичную страницу захвата лидов с презентационным блоком и формой заявки.

Важное изменение: Предыдущая страница с формой теперь становится админ-панелью и переносится на маршрут /admin (авторизация будет добавлена позже). Текущая задача — создать новую, оригинальную клиентскую страницу на корневом маршруте /.

Технологический стек
Сборка: npm + Vite (npm run build → dist/)
Фреймворк: на выбор (React, Vue или vanilla JS — главное, чтобы быстро собиралось и было легко поддерживать)
Стили: Tailwind CSS или SCSS-модули
Шрифты: Google Fonts, только округлые (без засечек) — например, Nunito, Nunito Sans, Rubik, Varela Round, Comfortaa (для акцентов)
Анимации: CSS-анимации или легковесные JS-анимации, без тяжёлых библиотек

Структура клиентской страницы (/)
1. Шапка-презентация (Hero-секция)
Назначение: представить бизнес, вызвать доверие, показать экспертизу.

Содержание (стилистика — по мотивам GitHub-профиля MatveiV, см. референс):
Заголовок: «Системный аналитик и Vibe-кодер с 20-летним опытом в FinTech»
Подзаголовок: «Торговая платформа, расчётная система или AI-агент под ваш процесс: Python + LLM-инструменты → MVP, который не нужно переписывать после первого релиза»
Краткое «Кто я»: 2-3 предложения о специализации — трансляция бизнес-требований в архитектурные решения и MVP, проектирование высоконагруженных систем, связка бизнес↔разработка.

Блок с ключевыми метриками/достижениями (в цифрах):
20+ лет опыта
30+ проектов в портфолио
5 AI-провайдеров в единой архитектуре
Технологии: Python, Go, LLM, Docker, PostgreSQL

Визуальные элементы: иконки технологий, аккуратная инфографика.
Кнопка CTA (призыв к действию): «Обсудить проект» / «Оставить заявку» — якорь, прокручивающий к форме ниже.

Референс стиля: GitHub-профиль MatveiV — структурированная презентация проектов, чёткая иерархия, технологии в тегах/бейджах, акцент на результат. Адаптировать в формат лендинга — более визуально, с анимациями, но сохранить деловую эстетику.

2. Блок «Проекты и результаты»
Назначение: портфолио-секция с уже сделанными проектами для укрепления доверия.

Содержание (взять из GitHub MatveiV, категории проектов):
Отобразить категории проектов в виде карточек-витрин с фильтрацией или аккордеоном:

Категория	Примеры проектов
AI / LLM / Боты	Agent-SystemAnalyst_MemoryFrameworks, Znaika_Neznaika_bot, MultiTools AI Agent Bot, ShortLongMemory Bots, Product MCP Bot, AI Client PDF Generator, RAG-Agent, Team Assistant Telegram Bot
Инфраструктура и бэкенд	Refactoring_MV (Python→Go), Loki-Grafana, CoinParser, MiniCRM
ML / Финансы	ML_Fin_Notebooks (бэктестинг, LSTM, TFT)

Каждая карточка:
Название проекта
Краткое описание (1-2 строки)
Стек технологий (бейджи/теги)
Ссылка на GitHub (опционально)

Визуал: сетка карточек, лёгкий ховер-эффект с подъёмом карточки, золотой акцент при наведении.

3. Форма заявки
Назначение: сбор лидов. Все поля, где требуется диапазон значений — ползунок (range slider).

Поля формы (соответствуют модели Lead):
Поле	Тип ввода	Особенности
Имя	text	first_name
Фамилия	text	last_name
Отчество	text (опционально)	middle_name
Контактные данные	text (email/phone)	contact_data
Ниша бизнеса	text / suggest	business_niche — можно с автодополнением
Размер компании	range slider	company_size — диапазон (напр., 1–10, 11–50, 51–200...)
Объём задачи	range slider	task_volume — условные единицы/часы
Роль	select (Сотрудник / Руководитель)	role
Информация о бизнесе	textarea	business_info
Бюджет	range slider	budget — диапазон из AdminData
Срок (дедлайн)	date picker / range slider (недели)	project_deadline
Тип задачи	select (динамический из AdminData)	task_type
Интересующий продукт	select (динамический из AdminData)	product_interest
Предпочтительный способ связи	radio-кнопки	preferred_contact_method
Удобное время связи	time picker / select	convenient_time
Комментарий	textarea (опционально)	comment

Особенности формы:
Ползунки стилизованы под общий дизайн (золотой трек, круглый бегунок).
Все селекты и диапазоны для ползунков загружаются динамически: GET /admin → получение AdminData (услуги, диапазоны бюджета, типы задач, продукты).

Валидация на фронте с красивыми ошибками.

При сабмите отправляется единый пакет { lead: {...}, behavior: {...} } на POST /leads.

4. Отслеживание поведения (Behavior Tracking)
Реализовать на клиентской странице:
Время на странице — time_spent_seconds
Клики — buttons_clicked (какие кнопки/элементы)
Зоны курсора — cursor_hover_zones (над какими секциями зависал)
Возвраты — return_count (через sessionStorage/localStorage)
При отправке формы пакет поведения прикрепляется к запросу.

Дизайн-система

Цветовая палитра:
Основной фон: светлый, чистый (#F9FAFB, #FFFFFF) — создать ощущение воздуха и пространства
Акцентный цвет: глубокий синий/индиго (#1E3A5F) для шапки и ключевых блоков
Золотые акценты (#D4AF37, #C9A84C) для кнопок, ползунков, иконок
Текст: тёмно-серый (#1F2937), заголовки — почти чёрный

Типографика:
Только округлые шрифты без засечек (sans-serif, rounded):
Заголовки: Nunito (жирный, 700-900)
Основной текст: Nunito Sans (400, 600)
Акценты, цифры: Comfortaa или Varela Round

Анимации:
В шапке — лёгкие плавающие геометрические фигуры (кружочки, мягкие звёздочки) с низкой прозрачностью, золотистого и голубого оттенков
При скролле — плавное появление секций (fade-in + slide-up)
Ховер-эффекты на карточках проектов и кнопках

Адаптивность:
Mobile-first: форма на мобильных в одну колонку, ползунки адаптированы под тач
Планшеты: 2 колонки
Десктоп: до 3 колонок в сетке проектов, форма в 2 колонки

Роуты бэкенда для интеграции
GET /admin — получить AdminData (список услуг, диапазоны бюджета, типы задач, продукты) для заполнения селектов и ползунков
POST /leads — отправить единый пакет { lead: {...}, behavior: {...} }

Итог: клиентская страница — это полноценный лендинг с шапкой-презентацией бизнеса, портфолио проектов (как на GitHub MatveiV, но визуально переработанное) и формой захвата лидов с ползунками и отслеживанием поведения. Админ-панель (/admin) остаётся отдельным маршрутом, авторизация будет добавлена позже.
