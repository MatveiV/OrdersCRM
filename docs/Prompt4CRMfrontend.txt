Промпт для AI-разработчика на создание фронтенда CRM:

Задача: Разработать фронтенд для CRM-системы (лендинг с формой заявки для "тёплых клиентов").

Технологический стек:
Сборка: npm (команда npm run build собирает всё в dist, готовое к деплою на сайт)
Настройка проекта через package.json с необходимыми скриптами (dev, build, preview)
Бандлер: Vite (быстрая сборка)
Стили: CSS-фреймворк на выбор (Tailwind CSS или чистый SCSS)
Шрифты: подключение через Google Fonts (элитные, премиальные шрифты)

Основной экран:
Пользователь сразу после захода на сайт видит главную страницу с формой заявки — без лишних кликов и переходов.

Дизайн и визуальный стиль:
Элитный, премиальный стиль — дорогой, минималистичный, с акцентами золота.
Цветовая палитра: глубокий тёмный фон (тёмно-синий/чёрный) + золотые акценты (#D4AF37, #FFD700, градиенты золота).
Типографика: шрифты Google Fonts — например, Cormorant Garamond (заголовки) + Montserrat/Inter (основной текст), кегль крупный, интерлиньяж воздушный.
Анимация фона: простенькая анимация с золотыми полупрозрачными кружочками и звёздочками, плавающими по экрану (CSS-анимация или легковесная JS-анимация, без тяжёлых библиотек).
Стекло-эффект (glassmorphism): сама форма на полупрозрачной подложке с blur-эффектом, тонкая золотая обводка.
Инпуты: стильные, с золотым бордером при фокусе, плавные переходы (transitions).
Кнопка отправки: золотая, с hover-эффектом свечения, текст контрастный.

Функциональные требования:

1. Загрузка административных данных:
При загрузке страницы фронтенд запрашивает GET /admin для получения списка услуг, диапазонов бюджета, типов задач и продуктов.
Динамически заполняет селекты и настраивает ползунок бюджета.

2. Отслеживание поведения пользователя:
Фиксируется время нахождения на странице (time_spent_seconds).
Отслеживаются клики по кнопкам/элементам (buttons_clicked).
Трекинг зон зависания курсора (cursor_hover_zones).
Подсчёт возвратов на страницу (return_count — через sessionStorage/localStorage).
Все метрики агрегируются на фронте.

3.Отправка данных:
При сабмите формы собирается единый пакет:
json
{
  "lead": { /* все поля формы */ },
  "behavior": { /* все метрики поведения */ }
}
Отправляется через POST /leads (бэкенд сам разложит по таблицам Lead и Behavior).

4.Валидация:
Обязательные поля помечены и валидируются на фронте.
Красивые сообщения об ошибках в стилистике дизайна.

5. Адаптивность:

Мобильная версия, планшеты, десктоп.

На мобильных анимация фона упрощается (производительность).

Структура проекта фронтенда:
frontend/
├── index.html
├── package.json
├── vite.config.js
├── src/
│   ├── main.js            — точка входа
│   ├── App.vue / App.jsx  — главный компонент
│   ├── components/
│   │   ├── AnimatedBackground.vue/jsx  — анимация фона (кружочки, звёздочки)
│   │   ├── GlassForm.vue/jsx           — форма с glassmorphism
│   │   └── FormField.vue/jsx           — переиспользуемый инпут
│   ├── api/
│   │   └── index.js        — функции для запросов к бэкенду
│   ├── tracking/
│   │   └── behavior.js     — модуль отслеживания поведения
│   ├── styles/
│   │   └── main.css        — глобальные стили, подключение Google Fonts
│   └── assets/

Сборка и деплой:
npm install — установка зависимостей.
npm run build — сборка в dist/.
Содержимое dist/ сразу готово к раздаче на сайте.

Роуты бэкенда, к которым идёт обращение:
GET /admin — получить AdminData (список услуг, диапазоны бюджета, типы задач, продукты)
POST /leads — отправить лид + поведение (единый пакет)

Итоговый результат: премиальная одностраничная форма захвата лидов с анимированным золотым фоном, отслеживанием поведения, динамическими полями и быстрой сборкой через npm.
