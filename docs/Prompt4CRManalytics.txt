Промпт для ИИ-агента на доработку аналитики заявок с интеллектуальным ранжированием и отображением в админ-панели:

Задача: Реализовать в админ-панели CRM интеллектуальное отображение заявок пользователей с автоматическим скорингом (определение «температуры» лида), приоритизацией, детальным просмотром каждой заявки и общей статистикой. Также предоставить SQL-код для генерации 10 тестовых заявок.

Часть 1. Модель данных и существующий эндпоинт

1.1. Таблица applications (существующая)
Предполагаемая структура (уточнить в Swagger UI):

Поле	Тип	Описание
id	INTEGER PK	Уникальный идентификатор заявки
first_name	VARCHAR	Имя
last_name	VARCHAR	Фамилия
middle_name	VARCHAR (nullable)	Отчество
contact_data	VARCHAR	Контактные данные (телефон/email)
business_niche	VARCHAR	Ниша бизнеса
company_size	VARCHAR	Размер компании
task_volume	VARCHAR	Объём задачи
role	VARCHAR	Роль (Сотрудник / Руководитель)
business_info	TEXT	Информация о бизнесе

Возвращает массив заявок. Используется для получения списка всех заявок в админ-панели.

2.2. Критерии оценки (балльная система)
Реализовать на бэкенде (Python, в отдельном модуле backend/core/scoring.py).

Таблица критериев:
Критерий	Макс. балл	Логика расчёта
Бюджет	25	Парсим budget (JSON или строка). Берём max значение: > 1 000 000 → 25, 500k-1M → 20, 100k-500k → 15, 50k-100k → 10, < 50k → 5, не указан → 0
Размер компании	20	"Крупная" / "Enterprise" / "500+" → 20, "Средняя" / "50-500" → 15, "Малая" / "10-50" → 10, "Микро" / "1-10" → 5, не указан → 0
Срок (дедлайн)	15	"Вчера" / "ASAP" / "до 1 недели" → 15, "1-2 недели" → 12, "2-4 недели" → 8, "1-3 месяца" → 5, "более 3 месяцев" / не указан → 0
Роль заполняющего	10	"Руководитель" / "CEO" / "Директор" → 10, "Менеджер" / "Team Lead" → 7, "Сотрудник" / "Специалист" → 4, не указан → 0
Объём задачи	10	Парсим или оцениваем: крупный → 10, средний → 6, малый → 3, не указан → 0
Ниша бизнеса	10	FinTech / Банки / Страхование → 10, IT / Телеком → 8, Ритейл / E-com → 6, Прочее → 4, не указана → 0
Полнота контактов	5	Есть телефон И email → 5, что-то одно → 3, только имя → 1
Комментарий	5	Есть заполненный комментарий (> 20 символов) → 5, короткий → 2, пустой → 0
Итого: максимум 100 баллов.

2.3. Определение температуры
Баллы	Температура	Лейбл	Цвет
70-100	hot	🔥 Горячий	Красный
40-69	warm	🌡 Тёплый	Оранжевый / Жёлтый
0-39	cold	❄️ Холодный	Синий / Серый

2.4. Рекомендации
Персональный менеджер: needs_personal_manager = true, если score >= 60 ИЛИ роль = "Руководитель"

Рекомендуемый отдел: определяется по business_niche и task_type:

FinTech / Торговая платформа → «FinTech-отдел»

AI / LLM / Боты → «AI-разработка»

Инфраструктура / Бэкенд → «Backend-отдел»

ML / Аналитика → «Data Science»

Прочее → «Общий отдел»

Разные критерии у горячих:
